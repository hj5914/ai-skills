# Skill README Guide

本文件用于规范 `skills/<skill-name>/README.md` 的结构，方便后续新增 skill 时保持仓库风格一致。

## 适用范围

每个面向 GitHub 展示或人工维护的 skill 都可以添加 README。

README 面向人类读者，帮助快速理解用途、安装、依赖和文件结构。Agent 执行规则仍以 `SKILL.md` 为准。

## 标准结构

```markdown
# Skill Name

一句话说明这个 skill 是什么。

补充 1-2 句话说明它解决的问题、依赖的工具或核心工作方式。

## 用途

- 这个 skill 能做什么。
- 主要能力按动词开头描述。

## 适合场景

- 适合使用这个 skill 的任务类型。
- 说明它最能带来价值的场景。

## 不适合场景

- 不应该使用这个 skill 的场景。
- 环境限制、风险限制或更简单替代方案。

## 安装/依赖

- 运行依赖。
- 安装方式。
- 如果没有额外依赖，明确写“无额外运行依赖”。

## 使用/验证

- 最小使用方式。
- 快速验证命令。
- 指向 `SKILL.md`，避免重复 agent 执行规则。

## 文件说明

```text
SKILL.md
  Agent 读取的主说明。

scripts/
  可执行脚本。

references/
  Agent 按需读取的参考资料。
```

## 备注

- 安全注意事项。
- 维护约定。
- 与其他 skill 或工具的关系。
```

## 编写原则

- README 给人看，`SKILL.md` 给 agent 执行。
- 不复制 `SKILL.md` 的完整流程，避免两份规则漂移。
- 结构统一，篇幅按 skill 类型调整。
- 工具型 skill 可以详细写安装、依赖、命令和验证。
- 流程型 skill 保持简短，重点说明用途、边界和文件结构。
- 有脚本或外部服务依赖时，必须写快速验证方式。
- 有安全风险时，必须在“备注”中说明。
